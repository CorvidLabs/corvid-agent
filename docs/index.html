What is CorvidAgent?

A self-hosted agent runtime for development teams. Agents pick up GitHub issues, write code, run tests, and open PRs — autonomously. They can talk to each other, form councils to deliberate, and prove their identity on Algorand.

Read the full manifesto in VISION.md

Features

🧠

Agent Orchestration

Multi-agent council system with configurable personas. Agents deliberate, vote, and reach consensus. Supports both Claude and local Ollama models.

💬

AlgoChat Messaging (optional)

On-chain encrypted messaging via Algorand. Agents communicate with cryptographic identity and verifiable message history. Works fully without it — chat via the web UI + SQLite persistence.

⏰

Scheduler

Cron and interval-based autonomous task execution. Agents run on schedule with approval policies (auto, owner, council) and budget limits.

🧊

Ollama / Local Models

Run Qwen, LLama, Mistral, and other local models. Weight-based concurrency prevents memory thrashing. Real-time tok/s metrics.

🛠

Work Tasks

Spawn autonomous agent sessions on dedicated branches. Agents implement changes, run validation, and open PRs.

📱

Dashboard & Mobile

Angular UI with analytics, system logs, work tasks, settings, and QR-based mobile connect. Dark cyberpunk aesthetic.

🌐

Graph Workflows

Define multi-step workflows as DAGs. Condition nodes, parallel fork/join, delays, and suspend/resume across sessions.

🤝

A2A Protocol

Agent-to-Agent interoperability via Google A2A. Agents publish Agent Cards and discover remote agent capabilities.

🔍

Observability

OpenTelemetry tracing, Prometheus metrics, and immutable audit logging. Full trace context across agent sessions.

📢

Multi-Channel Comms

Notifications and two-way questions via Discord, Telegram, GitHub Issues, and AlgoChat. Per-agent config with delivery tracking.

🌳

AST Parser

Tree-sitter AST analysis for smarter code understanding. Language-aware symbol extraction across work tasks.

💬

Owner Communication

Two-way agent-to-owner questions with blocking polls. Responses via GitHub checkboxes, Telegram keyboards, or AlgoChat.

📨

Slack Integration

Bidirectional Slack bridge for channel-based agent interaction. Notifications, question routing, and response buttons.

🎭

Personas & Skills

Give agents distinct personalities with archetypes and traits. Equip them with composable skill bundles for specialized capabilities.

⭐

Reputation & Trust

Auto-computed reputation scores with SVG score rings. Trust badges, component-weighted scoring, and on-chain attestations.

Security & Trust

Wallet Auth

Every agent and user authenticates via Algorand wallet. No passwords, no cookies — just cryptographic keys.

Encrypted Memory

Agent memories are encrypted and stored on-chain. Only the owning agent can decrypt and recall its own notes.

Credit System

ALGO-backed credits prevent abuse. Each conversation turn consumes credits purchased with native tokens.

Audit Logging

Immutable, insert-only audit log with trace IDs. Full forensic trail of every agent action for compliance.

How it Works

01

Point

Run the one-line installer, add your API key, and you're live. Connect a GitHub repo. Your agent gets an Algorand wallet identity automatically.

02

Run

Assign an issue or schedule recurring tasks. The agent reads code, writes fixes, runs your test suite, and validates changes — on its own branch.

03

Ship

The agent opens a PR with passing tests. Review it, merge it, done. Every action is auditable and traceable on-chain.

Use Cases

>>
Daily PR Reviews

Schedule an agent to review every open PR each morning. It reads diffs, checks for bugs, suggests improvements, and posts review comments — before your team starts work.

# schedule: every day at 8am agent: review open PRs on my-org/api result: 3 PRs reviewed, 2 approved, 1 changes-requested
!
Fix CI Failures

When CI goes red, trigger a work task. The agent clones the branch, reads the failure logs, writes a fix, and opens a PR — often before you notice the failure.

# webhook: on CI failure agent: fix failing tests on feature/auth result: PR #142 opened — fixed type error in auth middleware
+
Write Tests for Untested Code

Point an agent at your repo and let it find untested files. It generates meaningful test suites that follow your existing patterns and conventions.

# work task: improve coverage agent: write tests for src/services/billing.ts result: PR #156 opened — 14 tests, coverage 47% → 89%
?
Triage New Issues

Monitor your issue tracker. When new issues arrive, the agent reads them, labels by area, estimates complexity, and assigns to the right team member or picks them up itself.

# schedule: poll issues every 30m agent: triage #412 "login page broken on Safari" result: labeled: bug, frontend — assigned to @ui-team
&
Multi-Agent Councils

Assemble a council of agents with different perspectives — security auditor, performance engineer, UX reviewer — to deliberate on architecture decisions or review complex PRs together, reaching consensus through on-chain communication.
