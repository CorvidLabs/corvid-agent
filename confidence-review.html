Security: Council-Identified Vulnerability

LIVE FIX: postinstall Script Execution in Work Tasks

Identified by: Council deliberation on tiered permissions (6 agents, unanimous)

Vulnerability: bun install in work task worktrees ran without --ignore-scripts. A malicious dependency's postinstall hook could modify protected files (sdk-process.ts, spending.ts, etc.) before protected-file checks ran.

Impact: Complete bypass of the protected-file system via supply chain attack.

Fix: Added --ignore-scripts to all 4 bun install calls in server/work/service.ts (lines 143, 158, 383, 394).

Status: PATCHED in PR #91, CI green.

Known Issues & Next Steps

Ollama Tool Calling (v0.5.0)

Qwen3 models sometimes return tool calls as JSON text in the response body instead of using proper tool call format. Direct engine parses them but execution is inconsistent. Needs parser hardening or model-specific prompt engineering.

Kernel Extraction (Council Recommendation)

Council unanimously recommended extracting a "kernel" — permanently immutable files that no agent, council, or schedule can modify. Phase 0 of the tiered permissions roadmap. Breaks recursive governance loops.

PR #91 Pending Merge

Contains council_launch scheduler implementation + postinstall security fix. CI green. Needs merge by a different reviewer (branch protection).
